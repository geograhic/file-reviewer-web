# 智能文件复习系统 2.0 · Web 版部署说明

> 纯前端 PWA。数据**只存在用户本地浏览器（IndexedDB）**，不连接任何服务器、不消耗云端存储与算力。
> 与桌面版（Python `app.py` + pywebview）功能对等，但后端被完整移植为浏览器内运行的 JS。

---

## 1. 架构概览

```
reviewer-web/
├── index.html        # 入口，按 db.js → scheduler.js → api.js → app.js 顺序加载
├── app.js            # 前端 SPA（与桌面 web 版一致，仅新增 LocalAPI.install() 启动钩子）
├── style.css
├── db.js             # 数据层：浏览器用 IndexedDB，Node 测试环境自动降级为内存 Map
├── scheduler.js      # 调度算法：FSRS-Lite / SM-2 / Fixed（1:1 移植自 app.py，24 项交叉测试通过）
├── api.js            # 客户端“后端”：实现 app.py 的全部 /api/* 路由，install() 接管 window.fetch
├── locales/          # 多语言（内联字典，无需网络）
├── vercel.json       # Vercel 静态托管 + SPA 回退（保留静态资源与 /api/ 路径）
└── tests/            # Node 集成测试（无需浏览器）
```

**工作原理**：`app.js` 里的 `api()` 仍调用 `fetch("/api/...")`。启动时 `LocalAPI.install()` 给 `window.fetch` 打补丁——凡是 `/api/` 请求都被路由到 `api.js` 的本地路由器，由 IndexedDB 处理。前端代码**零改动**即可运行。

**文件与笔记的导入/打开**：
- 导入资料库 / 文件 → 浏览器原生文件夹/文件选择器（Chromium 用 File System Access API 持久句柄；其它浏览器用 `<input webkitdirectory>` / `<input multiple>` 仅取元数据），**不复制文件内容**。
- 打开文件 / 预览 → 用 `URL.createObjectURL` 在 `<iframe>/<img>/<video>` 内联预览，或触发下载。
- 导出 CSV / 便携包 / 个人资料 → 客户端生成 Blob 并触发下载，绝不离开本机。

## 2. 本地预览（无需部署即可验证）

任选一种静态服务器，在 `reviewer-web/` 目录内启动：

```bash
# 方式 A：Python（无需安装）
python -m http.server 8080

# 方式 B：Node
npx serve -l 8080 .
```

然后浏览器打开 `http://localhost:8080/`。

> 必须经由 HTTP 服务访问（不能用 `file://` 直接打开），否则 IndexedDB 与 fetch 补丁受浏览器安全策略限制。

## 3. 部署到 Vercel

### 方式一：Git 连接（推荐）
1. 将 `reviewer-web/` 作为独立仓库，或把整个项目连接后设置 **Root Directory = `reviewer-web`**。
2. Vercel 自动识别 `vercel.json`：
   - **Framework Preset**：Other / 静态
   - **Build Command**：留空（已是静态产物）
   - **Output Directory**：`.`（即 `reviewer-web` 根目录）
3. 点击 Deploy。

### 方式二：Vercel CLI
```bash
npm i -g vercel
cd reviewer-web
vercel            # 按提示部署；vercel.json 已就位
vercel --prod    # 上线生产
```

### SPA 回退说明
`vercel.json` 的重写规则为 `/((?!api/|.*\..*).*)` → `/index.html`：
- 含扩展名的路径（`.css/.js/.png` 等）→ 直接返回静态文件，不被重写。
- `/api/*` → 由浏览器内 `LocalAPI` 拦截，服务器不会真正收到（即便收到也会回退到 index.html，无副作用）。
- 其余无扩展名路径 → 回退到 `index.html`，保证刷新不 404。

## 4. 数据与隐私

- 所有资料、笔记、复习记录、社交资料均存于**当前浏览器**的 IndexedDB（按域名隔离）。
- **索引优先（只索引、不移动原始资料）**：默认只存储文件路径/标签/排程等**元数据**，文件字节**不入库**。Chromium 下用 File System Access API 持久保存「指向原文件的句柄」，打开/预览时按原路径按需读取；其它浏览器在本会话内记住所选文件，刷新后若需再次打开会提示重新选择原文件。仅「导出迁移包 / 离线副本」这类用户主动操作才会把文件内容写入包中。
- 因此正常使用的存储占用极小（每条记录仅几 KB 元数据），几乎不存在容量焦虑；真正的大文件始终留在用户自己的磁盘上。
- 换设备或换浏览器需使用「导出便携包 / 个人资料」，再在目标环境「导入」——与桌面版一致。
- 不收集任何遥测，不调用任何第三方接口。

## 6. 已知限制（与桌面版差异）

| 能力 | 桌面版 | Web 版 |
|------|--------|--------|
| 扫描本地任意文件夹 | `os.walk` 全磁盘 | 仅用户手动选择的文件夹/文件 |
| 打开原始文件 | `os.startfile` | 浏览器内预览 / 下载 |
| 数据持久化 | 本地 SQLite | 浏览器 IndexedDB（按域名） |
| 跨设备同步 | 手动导出/导入 | 手动导出/导入（同左） |

设计取舍：Web 版刻意「只索引、不移动原始资料」，符合「数据留在用户本地、不消耗云端资源」的初衷。
